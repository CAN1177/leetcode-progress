## 核心不变量
"左右指针指向的字符，在经过大小写转换和非有效字符过滤后，必须始终相等。"

## 攻击向量
最初尝试直接遍历比较，发现处理特殊字符很麻烦。随后改用双指针 `left` 和 `right` 从两头往中间扫。

## 模式 + 为什么这个模式适用
对撞指针。因为回文定义本身是对称的，从两端向中间汇合是验证对称性最自然且高效（$O(1)$ 额外空间）的方式。

## 最小骨架（对撞指针跳过逻辑版）
```javascript
while (left < right) {
    while (left < right && !isValid(s[left])) left++;
    while (left < right && !isValid(s[right])) right--;
    if (s[left].toLowerCase() !== s[right].toLowerCase()) return false;
    left++;
    right--;
}
```

## 触发信号
- 判断字符串是否对称（回文）。
- 数组或字符串需要从两头向中间聚合。
- 需要在 $O(1)$ 空间内处理字符串验证。

## 关键失误
在写内层跳过逻辑时，如果忘记再次判断 `left < right`，当字符串全是无效字符时，`left` 或 `right` 可能会越界。

## 口头讲解版本
就像检查一段话是不是倒着读也一样。我们用两个手指捏住头尾，如果指着的是空格或标点就挪开，只看字母和数字。只要指着的两个字母不一样，那就不是回文；如果两根手指碰头了都还没发现不一样的，那就是回文。
