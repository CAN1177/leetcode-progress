# 3. 无重复字符的最长子串 · 2026-04-17

## 元信息
- 难度：Medium
- 所属模式：滑动窗口 (可变窗口 - 求最长)
- 来源：LeetCode #3
- 语言：JavaScript

## Timebox
- 分配时长：20 分钟
- 实际用时：12 分钟
- 是否需要 LLM 提示：概念提示 (理解滑动窗口的呼吸感)

## 核心不变量
"窗口 `[left, right]` 始终包含一个无重复字符的集合。一旦右侧新入字符破坏此规则，左侧必须收缩直到重新符合规则。"

## 攻击向量
利用 `Set` 记录窗口内字符。`right` 指针不断右移探索，当遇到已存在于 `Set` 中的字符时，`left` 指针不断右移并从 `Set` 中删除元素，直到冲突解除。

## 模式 + 为什么这个模式适用
滑动窗口。本题寻找“最长连续子串”，且具有明显的单调性：窗口右移可能引入重复，左移则一定能消除重复。通过维护窗口状态避免了 $O(n^2)$ 的重复计算。

## 最小骨架（求最长滑动窗口版）
```javascript
let left = 0, right = 0, res = 0;
const window = new Set();
while (right < s.length) {
    let c = s[right];
    while (window.has(c)) {
        window.delete(s[left]);
        left++;
    }
    window.add(c);
    res = Math.max(res, right - left + 1);
    right++;
}
```

## 我的写法 vs 最优解（差异对照）
| 维度 | 我的写法 | 最优解 | 差异原因 |
| :--- | :--- | :--- | :--- |
| 时间复杂度 | O(n) | O(n) | 虽然有嵌套 `while`，但每个字符最多进出窗口各一次 |
| 空间复杂度 | O(min(m, n)) | O(min(m, n)) | `Set` 存储窗口字符，m 是字符集大小 |
| 优化点 | `Set` 逐个删除 | `Map` 记录索引直接跳转 | `Map` 法可将 `left` 直接移动到重复位置之后，减少删除操作 |

## 触发信号
- "最长/最短连续子串/子数组"
- 窗口内状态具有单调性（加一个数一定变大/变多，减一个数一定变小/变少）

## 关键失误
在 `while` 循环收缩窗口时，顺序必须是：先 `delete(s[left])`，再 `left++`。

## 口头讲解版本
想象一个会伸缩的袋子在字符串上爬。右手不停往袋子里抓字符，如果抓到的字符袋子里已经有了，左手就得不停从袋子另一头往外掏字符，直到那个重样的被掏出去为止。每抓一个新字符，都量一下袋子的长度，最长的那个就是答案。
