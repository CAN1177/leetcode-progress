## 核心不变量
"固定一个数 `nums[i]` 后，寻找满足 `nums[left] + nums[right] = -nums[i]` 的组合，且所有三元组及其内部元素必须去重。"

## 攻击向量
通过排序将无序搜索转为有序搜索，利用双指针从两端逼近目标值。

## 模式 + 为什么这个模式适用
对撞指针。由于数组已排序，`sum` 与指针移动方向存在单调关系（和太小则左移，和太大则右移）。配合外层循环固定一个变量，将 $O(n^3)$ 降为 $O(n^2)$。

## 最小骨架（3Sum 版）
```javascript
nums.sort((a, b) => a - b);
for (let i = 0; i < nums.length - 2; i++) {
    if (i > 0 && nums[i] === nums[i - 1]) continue; // 关键去重 1
    let left = i + 1, right = nums.length - 1;
    while (left < right) {
        let sum = nums[i] + nums[left] + nums[right];
        if (sum === 0) {
            res.push([nums[i], nums[left], nums[right]]);
            while (left < right && nums[left] === nums[left + 1]) left++; // 关键去重 2
            while (left < right && nums[right] === nums[right - 1]) right--; // 关键去重 3
            left++; right--;
        } else if (sum < 0) left++;
        else right--;
    }
}
```

## 我的写法 vs 最优解（差异对照）
| 维度 | 我的写法 | 最优解 | 差异原因 |
| :--- | :--- | :--- | :--- |
| 时间复杂度 | O(n²) | O(n²) | 一致 |
| 空间复杂度 | O(1) 或 O(log n) | O(1) 或 O(log n) | 空间主要由排序算法决定 |
| 去重处理 | 手动 `while` 循环跳过 | 同上 | 这是最高效的原地去重方式 |

## 触发信号
- 数组中找三个数（或更多）之和等于目标值。
- 需要处理结果去重。
- 可以先排序而不影响题目要求的场景。

## 关键失误
在实现 `left` 和 `right` 的去重时，需要先保存当前值或比较相邻值，且确保 `while` 内部依然有 `left < right` 边界检查。
