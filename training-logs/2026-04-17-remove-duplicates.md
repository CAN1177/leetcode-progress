# 26. 删除有序数组中的重复项 · 2026-04-17

## 元信息
- 难度：Easy
- 所属模式：双指针 (快慢指针)
- 来源：LeetCode #26
- 语言：JavaScript

## Timebox
- 分配时长：15 分钟
- 实际用时：8 分钟
- 卡点出现在第几分钟：3 (确认 slow 指针的具体含义是"最后一个有效元素"还是"下一个待填位置")
- 是否需要 LLM 提示：否

## 核心不变量
"慢指针 `slow` 及其左侧的数组部分（`nums[0...slow]`）始终是不重复且有序的有效区间。"

## 攻击向量
直接使用快慢指针同向遍历。快指针负责寻找新元素，慢指针负责维护有效边界。

## 模式 + 为什么这个模式适用
快慢指针。本题要求原地修改（In-place）且保持相对顺序，快慢指针可以在单次遍历中，通过快指针的“探测”和慢指针的“覆盖”完成所有操作。

## 最小骨架（快慢指针版）
```javascript
let slow = 0;
for (let fast = 1; fast < nums.length; fast++) {
    if (nums[fast] !== nums[slow]) {
        slow++;
        nums[slow] = nums[fast];
    }
}
return slow + 1;
```

## 我的写法 vs 最优解（差异对照）
| 维度 | 我的写法 | 最优解 | 差异原因 |
| :--- | :--- | :--- | :--- |
| 时间复杂度 | O(n) | O(n) | 一致 |
| 空间复杂度 | O(1) | O(1) | 一致 |
| 控制流 | `while` 循环 | `for` 循环 | 本质相同，`for` 循环在处理索引自增时更简洁 |
| 边界处理 | `if (nums.length === 0)` | 同上 | 处理空数组的必要检查 |

## 触发信号
- 原地修改有序数组/链表（In-place）。
- 删除重复项或特定值的元素。
- 保持元素的相对顺序。

## 关键失误
在返回结果时，要注意 `slow` 是索引，返回长度时需要加 1。另外，`fast` 从 1 开始可以减少一次不必要的比较。

## 口头讲解版本
想象你在整理一排书，里面有很多重复的。你用左手按住当前最后一本不重复的书，右手往后翻。只要右手翻到的书跟左手按住的那本不一样，你就把左手往后挪一个空位，把新书放进去。最后，左手的位置加 1 就是这排新书的总数。
