## 核心不变量
"在反转过程中，`prev` 始终指向已反转部分的头部，`current` 始终指向待处理部分的头部。核心在于‘先存后路、再断前路、最后推进’。"

## 攻击向量
利用双指针（迭代法）原地修改。通过暂存 `next` 节点防止断链。

## 模式 + 为什么这个模式适用
链表指针操作。反转链表是改变节点间 `next` 指向的最基础练习，通过迭代法可以达到 $O(1)$ 的空间复杂度，是链表题目的基石。

## 最小骨架（迭代反转版）
```javascript
let prev = null, curr = head;
while (curr) {
    let nextTemp = curr.next; // 1. 存后路
    curr.next = prev;         // 2. 反转
    prev = curr;              // 3. 推进 prev
    curr = nextTemp;          // 4. 推进 curr
}
return prev;
```

## 我的写法 vs 最优解（差异对照）
| 维度 | 我的写法 | 最优解 | 差异原因 |
| :--- | :--- | :--- | :--- |
| 时间复杂度 | O(n) | O(n) | 一致 |
| 空间复杂度 | O(1) | O(1) | 使用迭代法比递归法节省栈空间 |
| 健壮性 | 处理了空链表 | 同上 | `while (current !== null)` 自动兼容空链表 |

## 触发信号
- 链表顺序调整或反向遍历需求。
- 回文链表判断（反转后半部分）。
- 涉及链表结构的修改。

## 关键失误
在 `current.next = prev` 之前，必须先将 `current.next` 暂存，否则会丢失对后续链表的引用。
