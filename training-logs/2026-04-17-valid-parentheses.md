# 20. 有效的括号 · 2026-04-17

## 核心不变量
"最近一个进入栈的左括号，必须是第一个被对应的右括号匹配掉的。遍历结束时，栈必须为空（即所有债务清零）。"

## 攻击向量
利用 JS 数组的 `push` 和 `pop` 操作实现一个后进先出（LIFO）的栈。使用 `Map` 存储映射关系，遍历字符串时将左括号压栈，遇到右括号则弹出栈顶并校验。

## 模式 + 为什么这个模式适用
栈。本题具有明显的“就近匹配”特征，即当前遇到的右括号必须与离它最近的未匹配左括号配对。栈的后进先出特性完美契合这种“最近相关性”。

## 最小骨架（括号匹配版）
```javascript
const stack = [];
const map = { '(': ')', '[': ']', '{': '}' };
for (let char of s) {
    if (map[char]) {
        stack.push(char);
    } else {
        if (stack.length === 0 || map[stack.pop()] !== char) {
            return false;
        }
    }
}
return stack.length === 0;
```

## 我的写法 vs 最优解（差异对照）
| 维度 | 我的写法 | 最优解 | 差异原因 |
| :--- | :--- | :--- | :--- |
| 时间复杂度 | O(n) | O(n) | 一致 |
| 空间复杂度 | O(n) | O(n) | 最坏情况下所有括号都是左括号，全部入栈 |
| 容错性 | 严谨 (显式检查 `stack.length === 0`) | 同上 | 处理了 `")"` 或 `"("` 等非闭合情况 |

## 触发信号
- "就近原则" 的匹配问题。
- 需要撤销操作（Undo）。
- 路径化简（如 `/a/b/../c`）。
- 表达式求值或语法解析。

## 关键失误
- 遍历结束后，如果栈不为空，说明还有未匹配的左括号，这种情况容易漏掉 `return stack.length === 0`。
- 遇到右括号弹出前，没有检查栈是否为空（防止 `pop()` 返回 `undefined` 导致误判）。

## 口头讲解版本
就像玩消消乐。看到左括号就把它扔进桶里（压栈）；看到右括号，就从桶的最上面抓出一个来，看看是不是刚好配对。如果配得上，两个都消失；如果配不上，或者桶里根本没东西了，那就说明这串符号是乱套的。最后桶里必须干净如初，才算完美匹配。
