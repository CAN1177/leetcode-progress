# 153. 寻找旋转排序数组中的最小值 (Find Minimum in Rotated Sorted Array)

- **日期**: 2026-04-22
- **模式**: [Binary Search](../patterns/binary-search.md)
- **难度**: 中等
- **状态**: 复习完成 (边界收缩逻辑已稳固)

## 训练记录

### 1. 核心不变量
在旋转排序数组中，最小值右侧的所有元素都小于或等于原数组的最大值，而最小值左侧的所有元素都大于或等于原数组的最小值。通过与 `nums[right]` 比较，可以百分之百确定最小值落在 `mid` 的左侧还是右侧。

### 2. 触发信号
- 旋转排序数组。
- 寻找转折点、极值或断层点。
- 只有 $O(\log n)$ 时间限制。

### 3. 关键感悟
- **基准物的重要性**：在处理寻找转折点的二分时，与 `right` 比较通常比与 `left` 比较更具普适性，因为它可以直接区分 `mid` 是否已经越过了旋转点。
- **收缩边界的精细化**：
    - `left = mid + 1`：当确定 `mid` 处在“左侧较大升序段”时，可以大胆跳过。
    - `right = mid`：当 `mid` 可能就是最小值时，必须保留在区间内，防止丢失解。
- **退出条件的配合**：`while (left < right)` 配合 `right = mid` 是处理这种“保留型”二分的标准套路，最终 `left` 即为结果。
