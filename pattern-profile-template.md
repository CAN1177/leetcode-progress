# 模式深度档案模板

> 本文件是**模板种子**。真实档案写在外部记录仓库的 `patterns/<pattern-slug>.md` 里，每个模式一个文件。
>
> 这张表回答一个问题：**下次 Agent 给我出一道新题、而我需要 2 秒内判断"这是不是我已经会的模式"时，我能从哪里读到判据？**

---

## 模板

```markdown
# <模式中文名> / <English name>

## 一句话定义
<例如："在连续子区间上维护一个单调变化的不变量，用左右指针调整区间边界。">

## 核心不变量
<剥到最后这个模式到底在守护哪个常量关系？一句话。>

## 触发信号（气味清单）
题目里出现以下任一线索，就应该立刻想到这个模式：

- 线索 1：<具体到一眼识别。例："最长/最短连续子串/子数组">
- 线索 2：<例："窗口内满足某条件的最多字符数">
- 线索 3：...

## 反触发信号（看起来像但其实不是）
- 例："区间不连续 → 不是滑动窗口，是子序列 DP"
- 例："要求所有子区间都处理 → 不是单指针滑窗，是前缀和"

## 最小骨架（能背下来）
```
<5-10 行脱离任何具体题的伪代码骨架，下次遇到直接往里填条件>
```

## 常见变体
| 变体 | 关键差异 | 代表题 |
| :--- | :--- | :--- |
| 定长窗口 | 窗口长度固定 | 大小为 K 的最大和 |
| 可变窗口（求最长） | 满足条件时扩，不满足时收 | 无重复字符最长子串 |
| 可变窗口（求最短） | 满足条件时收，不满足时扩 | 长度最小的子数组 |

## 我做过的题（按时间倒序）
| 日期 | 题目 | 难度 | 角色 | Log |
| :--- | :--- | :--- | :--- | :--- |
| 2026-04-13 | 无重复字符最长子串 | Medium | 首次引入 | [log](../training-logs/2026-04-13-longest-substring.md) |

> "角色"字段：`首次引入` / `巩固` / `变体` / `压力重练`。
> Dominik 的观察：Easy 题常是**首次引入**，Medium 往往只是**变体/巩固**——这一列可以显式标注。

## 踩过的坑
- <例："收缩窗口时忘了同步更新 map 计数 → 死循环">
- <例："把 `while` 条件写成 `if` → 漏 case">

## 下一道值得挑战的题
<一道即可>

## 最近复述（Verbalization 版本）
<用生活化语言 2-3 段把这个模式讲清，就像讲给不懂算法的朋友。压力下能从"讲过的"重建代码。>

## 状态
- 当前状态：`未开始` / `引入中` / `学习中` / `待复习` / `已稳固`
- 最后一次实战：<日期>
- 下一次复习建议：<日期>（一般距上次 1 周 → 1 月 → 3 月）
```

## 为什么要有这份档案（和训练日志的区别）

| 维度 | `training-logs/*.md` | `patterns/*.md`（本档案） |
| :--- | :--- | :--- |
| 粒度 | **单题一份** | **单模式一份** |
| 生命周期 | 写完就基本不改 | 每练到同模式的新题就回来追加 |
| 读的时机 | 回顾某次训练 | 开始新题前扫"我会哪些模式" |
| 最有价值的内容 | 当时的失误 + 差异对照 | **触发信号 + 最小骨架** |

---

## 灵感来源

本档案结构参考 [My Google Recruitment Journey](http://blog.dominikrudnik.pl/my-google-recruitment-journey-part-1) 里的三个观察：

1. 算法学习的真正产出是**模式识别**，不是单题 AC
2. **触发信号**比代码本身更难学，但更值钱
3. **最小骨架** + **用自己风格重写**是把概念刻进肌肉记忆的关键
