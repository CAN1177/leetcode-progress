# 滑动窗口 / Sliding Window

## 一句话定义
在序列（数组或字符串）上通过两个指针（左右边界）维护一个满足特定条件的连续区间，并在移动中动态更新其状态。

## 核心不变量
- **最长子区间**：当窗口满足条件时，右边界 `right` 扩张；当破坏条件时，左边界 `left` 收缩。
- **最短子区间**：当窗口不满足条件时，右边界 `right` 扩张；当满足条件时，左边界 `left` 收缩寻找最优解。

## 触发信号（气味清单）
题目里出现以下任一线索，就应该立刻想到这个模式：
- 线索 1：求“最长”或“最短”的连续子串、子数组。
- 线索 2：要求子区间的元素满足某种统计条件（如和大于 K、包含所有目标字符、无重复等）。
- 线索 3：题目给出的数据规模要求 $O(n)$ 时间复杂度。

## 反触发信号（看起来像但其实不是）
- 例：寻找非连续的子序列组合 → 通常是 DP 或回溯。
- 例：求所有子区间的某些属性 → 可能是前缀和（Prefix Sum）。

## 最小骨架（求最长窗口版）
```javascript
let left = 0, right = 0;
while (right < s.length) {
    let a = s[right]; // 1. 入窗
    right++;
    // 更新状态...
    
    while (windowState.notValid) { // 2. 不合法时收缩
        let d = s[left];
        left++;
        // 更新状态...
    }
    // 3. 记录结果
    res = Math.max(res, right - left);
}
```

## 常见变体
| 变体 | 关键差异 | 代表题 |
| :--- | :--- | :--- |
| 定长窗口 | 窗口大小固定，不需 `while` 收缩 | 滑动窗口最大值 |
| 不定长窗口 (最长) | 违规才收缩，收缩后记录结果 | 无重复字符的最长子串 |
| 不定长窗口 (最短) | 合格才收缩，收缩中寻找最小 | 最小覆盖子串 |

## 踩过的坑
- `right - left` 还是 `right - left + 1`？取决于 `right` 是否已经先 `++`。
- 忘记同步更新窗口内的状态（如计数器、累加值）。

## 最近复述（Verbalization 版本）
右指针一直在探索边界，像个贪吃的蛇，如果吃到了违规的东西（打破了不变量），左指针就要开始赎罪（吐掉），直到窗口重新变回合法状态。
