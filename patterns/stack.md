# 栈 / Stack

## 一句话定义
一种后进先出（LIFO）的线性数据结构，通过“最近入栈”的元素来处理“最近相关”的问题。

## 核心不变量
- **LIFO**：最后进入的元素必须最先处理。
- **最近相关性**：处理当前元素时，如果需要依赖“刚才那个元素”的状态，栈是首选。

## 触发信号（气味清单）
题目里出现以下任一线索，就应该立刻想到这个模式：
- 线索 1：对称性或嵌套结构的匹配（如括号、HTML 标签、XML 解析）。
- 线索 2：需要撤销、回溯或寻找“上一个”状态。
- 线索 3：寻找数组中每个数左侧或右侧“第一个比它大/小”的数（触发单调栈）。
- 线索 4：深度优先遍历（DFS）的非递归实现。

## 反触发信号（看起来像但其实不是）
- 例：公平排队（先来后到） → 这是队列（Queue，FIFO）。
- 例：需要随机访问任意位置的元素 → 数组更合适。

## 最小骨架（基础栈操作）
```javascript
const stack = [];
for (let item of data) {
    if (shouldPush(item)) {
        stack.push(item);
    } else if (shouldPop(item)) {
        const top = stack.pop();
        // 处理匹配或计算逻辑...
    }
}
return stack.length === 0;
```

## 踩过的坑
- `pop()` 操作前未检查 `isEmpty` (JS 中会返回 `undefined`)。
- 遍历结束后忘记检查 `stack.length === 0`。
- 单调栈在 `while` 循环弹出时，忘记处理最终剩余在栈中的元素。
