# 链表 / Linked List

## 一句话定义
通过节点的指针（next）在不连续的内存空间中建立逻辑连接的线性数据结构。

## 核心不变量
- **指针接力**：操作链表时，始终保证对当前节点的引用，并在修改 next 指向前备份后继节点。
- **哑节点 (Dummy Node)**：在增删头节点时，使用 dummy 指向 head，统一操作逻辑。

## 触发信号（气味清单）
题目里出现以下任一线索，就应该立刻想到这个模式：
- 线索 1：数据结构本身就是单链表或双链表。
- 线索 2：要求在 $O(1)$ 空间内对序列进行重组。
- 线索 3：需要频繁在头部或中间进行增删操作。

## 反触发信号（看起来像但其实不是）
- 例：需要频繁通过索引随机访问元素 → 链表 $O(n)$ 不如数组 $O(1)$。
- 例：需要反向遍历单链表 → 除非先反转或转为双链表。

## 最小骨架（迭代遍历版）
```javascript
let curr = head;
while (curr) {
    // 处理逻辑...
    curr = curr.next;
}
```

## 最小骨架（反转版）
```javascript
let prev = null, curr = head;
while (curr) {
    let next = curr.next;
    curr.next = prev;
    prev = curr;
    curr = next;
}
return prev;
```

## 踩过的坑
- 断链 (Dangling pointer)：修改 next 时没存后路。
- 忘记更新 prev 和 current 导致死循环。
- 空链表或只有一个节点的边界处理。
